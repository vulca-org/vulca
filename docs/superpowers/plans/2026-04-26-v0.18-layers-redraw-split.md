# Vulca v0.18.0 Implementation Plan — `layers_redraw` Default Flip + `layers_split` Multi-Instance

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.18.0 with two changes: (a) `layers_redraw` defaults flipped to safer values (output to `<layer>_redrawn.png`, cream background, alpha preserved); (b) `layers_split` orchestrated mode supports multi-instance entities via per-entity `multi_instance: true` plan flag, returning N flat sibling layers.

**Architecture:** Change A is a default flip + 1 new opt-out kwarg (`in_place: bool`); existing `output_layer_name`/`background_strategy`/`preserve_alpha` machinery (added in v0.17.14) is reused. Change B adds an additive `multi_instance` flag through three layers (plan JSON → orchestrator → `detect_all_bboxes()` → entity loop), with `dict[label, list[bbox]]` return type discriminated by `isinstance(..., list)` at the call site. Backward compatibility preserved end-to-end.

**Tech Stack:** Python 3.11, MCP server framework (FastMCP), Grounding DINO + SAM (transformers==4.49 pinned), pytest, pillow.

**Spec:** `docs/superpowers/specs/2026-04-26-v0.18-layers-redraw-split-design.md` (committed at `564243f9`)

```python
def _nms_bboxes(detections, iou_threshold=0.5, keep_n=None):
    """Non-max suppression on (bbox, score, phrase) tuples.

    Keeps higher-score detections; removes lower-score ones that overlap >threshold.
    If keep_n is given, caps result to top-N (after NMS dedup).
    """
    detections = sorted(detections, key=lambda d: -d[1])
    kept = []
    for d in detections:
        bbox, score, phrase = d
        is_dup = False
        for k in kept:
            if _iou(bbox, k[0]) > iou_threshold:
                is_dup = True
                break
        if not is_dup:
            kept.append(d)
            if keep_n is not None and len(kept) >= keep_n:
                break
    return kept
```

def detect_all_bboxes(
    dino_proc, dino_model, device, img_pil, labels, threshold=0.15,
    multi_instance=None,                # dict[label, max_n] or None
    multi_instance_box_threshold=0.25,  # lower threshold for multi-instance labels only
):
    """Single Grounding DINO pass with all labels jointly.

    Returns:
        dict label -> (bbox, score, matched_phrase) for single-instance labels;
        dict label -> list[(bbox, score, phrase)] for labels in multi_instance dict.
    """
    multi_instance = multi_instance or {}
    # If any multi-instance labels are requested, use the lower threshold for the
    # detection pass (recall is the priority). Single-instance labels accept the
    # extra noise; we filter at the per-label step.
    effective_threshold = (
        multi_instance_box_threshold if multi_instance else threshold
    )
    text = ". ".join(labels) + "."
    inputs = dino_proc(images=img_pil, text=text, return_tensors="pt").to(device)
    with torch.no_grad():
        outputs = dino_model(**inputs)
    results = dino_proc.post_process_grounded_object_detection(
        outputs, inputs.input_ids,
        threshold=effective_threshold, text_threshold=0.15,
        target_sizes=[img_pil.size[::-1]]
    )[0]

    # Existing per-label loop runs as before, but now collects ALL matching
    # detections per label (instead of overwriting with last/best). Then the
    # final step picks top-1 OR top-N depending on multi_instance dict.
    assigned_raw: dict[str, list[tuple]] = {lbl: [] for lbl in labels}
    det_scores = results["scores"].cpu().numpy()
    det_boxes = results["boxes"].cpu().numpy()
    det_labels = results.get("text_labels", results.get("labels", []))
    if hasattr(det_labels, "cpu"):
        det_labels = det_labels.cpu().numpy().tolist()

    for i in range(len(det_boxes)):
        phrase = det_labels[i] if isinstance(det_labels[i], str) else str(det_labels[i])
        phrase_words = set(phrase.lower().split())
        # Find best matching label by word overlap (preserve existing matching logic)
        best_label = None
        best_overlap = 0
        for lbl in labels:
            lbl_words = set(lbl.lower().split())
            overlap = len(phrase_words & lbl_words)
            if overlap > best_overlap:
                best_overlap = overlap
                best_label = lbl
        if best_label is None:
            continue
        bbox = [int(x) for x in det_boxes[i].tolist()]
        score = float(det_scores[i])
        assigned_raw[best_label].append((bbox, score, phrase))

    # Final shape: tuple for single-instance, list for multi-instance
    assigned = {}
    for lbl, dets in assigned_raw.items():
        if not dets:
            continue
        if lbl in multi_instance:
            max_n = max(1, int(multi_instance[lbl]))
            kept = _nms_bboxes(dets, iou_threshold=0.5, keep_n=max_n)
            assigned[lbl] = kept            # list form
        else:
            kept = _nms_bboxes(dets, iou_threshold=0.5, keep_n=1)
            assigned[lbl] = kept[0]          # tuple form (backward compat)
    return assigned
```

class TestMultiInstanceL4:
    """L4: real DINO + SAM on lanterns fixture. CI-skipped via marker."""

    @pytest.mark.l4_local
    def test_real_dino_sam_on_lanterns_fixture(self):
        """6-lantern fixture → ≥3 instances detected with sam_score>0.5, no overlap."""
        from scripts import claude_orchestrated_pipeline as cop

        plan = {"object_entities": [
            {"label": "red paper lanterns", "z_index": 1, "multi_instance": True}
        ]}
        result = cop.pipeline(
            image_path="tests/fixtures/multi_instance/lanterns_6.jpg",
            plan=plan,
            multi_instance_labels={"red paper lanterns": 8},
        )
        layers = [l for l in result["layers"]
                  if l["label"] == "red paper lanterns"]
        assert len(layers) >= 3, \
            f"DINO should detect ≥3 lanterns, got {len(layers)}"

        for l in layers:
            assert l["sam_score"] > 0.5, \
                f"sam_score too low for {l['name']}: {l['sam_score']}"

        # IoU check
        def iou(a, b):
            ax1, ay1, ax2, ay2 = a
            bx1, by1, bx2, by2 = b
            inter_w = max(0, min(ax2, bx2) - max(ax1, bx1))
            inter_h = max(0, min(ay2, by2) - max(ay1, by1))
            inter = inter_w * inter_h
            area_a = (ax2 - ax1) * (ay2 - ay1)
            area_b = (bx2 - bx1) * (by2 - by1)
            union = area_a + area_b - inter
            return inter / union if union > 0 else 0.0

        bboxes = [l["bbox"] for l in layers]
        for i, b1 in enumerate(bboxes):
            for b2 in bboxes[i+1:]:
                assert iou(b1, b2) < 0.5, \
                    f"instances should not overlap heavily: IoU={iou(b1, b2)}"

        report = next(r for r in result["detection_report"]["per_entity"]
                      if r["label"] == "red paper lanterns")
        assert "multi_instance_degraded" not in report.get("quality_flags", [])
```

## v0.18.0 — 2026-04-29

### Breaking changes

#### `layers_redraw` defaults flipped

The default behavior of `layers_redraw` is now non-destructive and
hallucination-resistant. Three defaults flipped:

| Parameter | v0.17.x default | v0.18.0 default |
|---|---|---|
| Output path | in-place overwrite | `<layer>_redrawn.png` (input untouched) |
| `background_strategy` | `"transparent"` | `"cream"` |
| `preserve_alpha` | `False` | `True` |

A new parameter `in_place: bool = False` lets callers explicitly request
the legacy in-place write. To restore v0.17.x behavior verbatim:

```python
layers_redraw(
    artwork_dir, layer="lanterns", instruction="...",
    in_place=True,
    background_strategy="transparent",
    preserve_alpha=False,
)
```

Why: v0.17.14 introduced the opt-in defenses but left legacy as the
default, leaving the trap (silent input destruction + scene hallucination
on alpha-sparse layers) as the path of least resistance. γ Scottish
Part 2 showcase (2026-04-25) confirmed the trap is easy to fall into.
v0.18 makes the safe path the default; the unsafe path requires explicit
opt-out.

### New features

#### `layers_split` orchestrated mode supports multi-instance entities

Plan JSON entities now accept an optional `multi_instance: true` flag.
When set, Grounding DINO returns up to 8 bboxes for that label (instead
of top-1), each segmented by SAM independently. Output is N flat sibling
layers named `<label>_0..N-1` sorted by sam_score descending.

```json
{
  "object_entities": [
    { "label": "red paper lanterns", "z_index": 2, "multi_instance": true }
  ]
}
```

Edge cases:
- DINO returns 0 bboxes → 0 layers + `quality_flags: ["multi_instance_no_detection"]`
- DINO returns 1 bbox → 1 layer named `<label>` (no suffix) + `quality_flags: ["multi_instance_degraded"]`
- DINO returns >8 bboxes → top-8 by score retained
- z_index of subsequent entities auto-pushed by `(N-1)` to make room

Closes the multi-instance gap deferred from v0.17.13. Plan JSON schema
extension is additive; existing plans without the flag are unaffected.

### Internal

- New L4 fixture `tests/fixtures/multi_instance/lanterns_6.jpg` (γ Scottish lanterns crop) anchored as multi-instance regression baseline.
- New `l4_local` pytest marker for local-only model-weight tests; CI auto-skips.
```
