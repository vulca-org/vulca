# v0.22 Target-Aware Mask Refinement Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add target-aware mask refinement so broad flower-like masks are split into useful child patches before redraw.

**Architecture:** A new `vulca.layers.mask_refine` module diagnoses layer/intent geometry and returns refined full-canvas child masks. `redraw_layer()` calls it before route selection and, when refinement applies, reuses the v0.21 crop-aware masked edit path per child. `redraw_quality.py` receives refinement-aware gates and advisory fields explain every automatic decision.

**Tech Stack:** Python, Pillow, NumPy, existing Vulca layer manifest/redraw/provider abstractions, pytest.

---

## File Structure

- Create `src/vulca/layers/mask_refine.py`: target profile inference, diagnostics, child-mask extraction, box merging.
- Modify `src/vulca/layers/redraw.py`: call refinement before route choice, redraw refined child masks, include advisory metadata.
- Modify `src/vulca/layers/redraw_quality.py`: expose target-aware granularity metrics and failures.
- Add `tests/test_mask_refine.py`: deterministic mask refinement unit tests.
- Add `tests/test_layers_redraw_refinement.py`: redraw integration tests using a recording edit provider.
- Keep existing tests unchanged unless public behavior explicitly changes.
