## P0 ‚Äî Must fix before v0.13.1 ships

### P0.1 ‚Äî `layers retry` defaults `--tradition` to `chinese_xieyi` and silently mis-regenerates

**Severity:** Silent data corruption.

**Sites:**
- `src/vulca/cli.py:262-264` ‚Äî `layers retry` argparse default
- `src/vulca/mcp_server.py:1517-1518` ‚Äî `vulca_layers_retry` MCP tool default
- `src/vulca/layers/retry.py:102-111` ‚Äî uses passed `tradition` to re-derive anchor / canvas / keying
- `src/vulca/layers/manifest.py:37-72` ‚Äî manifest schema does NOT persist `tradition`
- `src/vulca/layers/artifact.py:55-63` ‚Äî artifact.json DOES record tradition (asymmetric)

**Why it matters:** A user calling `vulca layers retry artifacts/<run>/` against an `islamic_geometric` or `brand_design` artifact without explicitly passing `--tradition islamic_geometric` will get the xieyi prompt and keying strategy applied silently. The layered PNGs are overwritten. There is no warning, no diff, no opt-in.

**Fix (preferred):** Persist `tradition` into `manifest.json` (bump schema to v3.1 ‚Äî backwards-compatible read; missing field treated as "unknown" and forces explicit `--tradition`). `retry_layers` reads it from manifest. CLI/MCP `--tradition` becomes optional and only required when manifest lacks it. If user passes `--tradition` AND it disagrees with manifest, error out (do not silently override).

**Fix (minimum acceptable):** Make `--tradition` a *required* argument on both CLI and MCP. No default. This is uglier but eliminates the foot-gun in one commit.

**Verification:** Add a test that calls `retry_layers` on a non-xieyi manifest without passing tradition and asserts the call fails (or reads the right value from manifest).

### P0.2 ‚Äî `LayerInfo.position` / `coverage` are dunder-private; retry drops them

**Severity:** Latent retry-correctness bug. Retries silently use the weaker default spatial block AND get a different cache key from the original run, so cache hits become impossible after a retry.

**Sites:**
- `src/vulca/layers/prompt.py:75-87` ‚Äî `parse_v2_response` sets `_position` / `_coverage` via `setattr` on the dataclass
- `src/vulca/layers/types.py:13-40` ‚Äî `LayerInfo` has no `position` / `coverage` fields
- `src/vulca/layers/manifest.py:90-111` ‚Äî manifest load does not round-trip them
- `src/vulca/layers/retry.py:97-122` ‚Äî `retry_layers` does not pass `positions` / `coverages` to `layered_generate`
- `src/vulca/layers/layered_prompt.py:37-54` ‚Äî falls back to weak default spatial block when position is missing
- `src/vulca/layers/layered_generate.py:109-125` ‚Äî these inputs feed `build_cache_key`

**Fix:**
1. Promote `position: str = ""` and `coverage: str = ""` to first-class fields on `LayerInfo`.
2. Update `parse_v2_response` to set them as fields (not via `setattr` of dunder names).
3. Persist into `manifest.json` per layer.
4. `manifest.load_layers()` round-trips them.
5. `retry_layers` threads them back into `layered_generate`.
6. Audit any other site that does `getattr(layer, "_position", ...)` and switch to attribute access.

**Verification:** A retry test that asserts the second-run cache key matches the first-run cache key when only the layer set is filtered (no spec/canvas/key changes).

### P0.3 ‚Äî Retry drops `width` / `height`, regenerated layers get `0√ó0` cache key

**Severity:** Retry bypasses cache + produces wrong-dimension layers.

**Sites:**
- `src/vulca/layers/retry.py:113-122` ‚Äî does not read `width`/`height` from manifest
- `src/vulca/layers/manifest.py:39-40` ‚Äî manifest DOES persist dimensions
- `src/vulca/pipeline/nodes/layer_generate.py:144-168` ‚Äî original run threads live `ctx["size"]`
- `src/vulca/layers/layered_generate.py:114-125` ‚Äî `build_cache_key` uses `(w, h)`

**Fix:** `retry_layers` reads `width`/`height` from the loaded manifest and passes them to `layered_generate`. Same shape as P0.2, different field.

**Verification:** Retry test on a non-default-size artifact (e.g. 768√ó1024) that asserts the regenerated layer's PNG dimensions match.

### P0.4 ‚Äî Cache key has no `key_strategy_version`; algorithm-only fixes silently reuse stale PNGs

**Severity:** Latent correctness on every future keying-algorithm change.

**Sites:**
- `src/vulca/layers/layered_cache.py:21-40` ‚Äî `build_cache_key` hashes static schema `0.13` + `key_strategy` (class name) + canvas + dims
- `src/vulca/layers/layered_generate.py:114-125` ‚Äî passes `keying.__class__.__name__`

**Why it matters:** This is not theoretical. The v0.13.1 NTH commit `b7a4472` (ChromaKeying linear-RGB distance) is exactly the kind of algorithm fix that doesn't change the class name. Any user who upgrades from v0.13.0 to v0.13.1 with cached PNGs from a colored-tradition run will silently reuse the wrong-alpha PNGs. The cache will swallow the fix.

**Fix:** Add a `cache_version: ClassVar[int] = 1` (or a `cache_signature() -> str`) to each `KeyingStrategy` subclass. `build_cache_key` includes both `class_name` and `cache_version`. Bump `ChromaKeying.cache_version` to 2 in the same commit as the linear-RGB fix lands (or in a follow-up commit on PR #2 that bumps it to 2 and notes "invalidates pre-linear-RGB caches").

**Verification:** Unit test that two strategies with the same class name but different `cache_version` produce different cache keys.

## P2 ‚Äî v0.13.2 backlog (do NOT block v0.13.1)

These are real but lower severity. Track in a follow-up issue or v0.13.2 plan.

### Telemetry & API hygiene
- **`attempts` telemetry is a lie** ‚Äî `generate_one_layer` always reports `attempts=1` even on failure. Either remove the field as premature, or wire a single in-process retry on `generation_failed` (NOT `validation_failed`, that's deterministic). `src/vulca/layers/layered_generate.py:138,151,156`
- **Plan Task 17 drift** ‚Äî plan promised "per-layer retry budget of 2"; shipped code does zero in-process retries. The external `layers retry` CLI is not a substitute (it's an operator command, not an automatic budget). Update plan or implement budget. `docs/superpowers/plans/2026-04-08-layers-redesign.md`
- **`LayerFailure.reason` is a bare string** ‚Äî CLI and MCP downstream string-match on `"validation_failed"` / `"generation_failed"`. Promote to `Literal[...]` or `Enum`. `src/vulca/layers/layered_generate.py:197-207`

### Robustness
- **Tier-2 keying dotted-path loader has no error wrapping** ‚Äî typos like `"vulca.foo:bar"` raise raw `ModuleNotFoundError` instead of a keying-specific `ValueError`. `src/vulca/layers/keying/__init__.py:41-47`
- **`_apply_alpha` silently resizes shape mismatches** ‚Äî masks a real bug where keying ran on a differently-shaped image than the provider returned. Should `assert` instead of bilinear-fixup. `src/vulca/layers/layered_generate.py:83-88`
- **`layered_cache.py` has no temp-write-and-rename** ‚Äî two concurrent retry processes on the same artifact dir can corrupt a `.png` via partial write. Not a blocker for single-user CLI but matters for the MCP server path. `src/vulca/layers/layered_cache.py:44-51`
- **`_provider_supports_native` hardcodes `"mock"` string match** ‚Äî should ask the provider via a Protocol method (`supports_raw_rgba()`). Future `mock_png` or user-registered `mock_*` providers will silently misroute. `src/vulca/pipeline/nodes/layer_generate.py:83-86`

### Manifest schema
- **`layer_extras` flat spread risks key collision** ‚Äî uses `**` spread before explicit fields, so a future extras key like `"id"` would silently overwrite. Whitelist-spread or nest under `"extras"`. `src/vulca/layers/manifest.py:51-53`
- **Retry drops non-validation warnings** ‚Äî original tradition-layerability discouraged warning is lost when manifest is rebuilt on retry. Preserve from prior manifest. `src/vulca/layers/retry.py:210-222`

### Performance / efficiency
- **Cache write double-encodes** ‚Äî layer is saved to disk on line 121 then re-encoded via PIL ‚Üí bytes for cache. Either re-read the file or write cache first then disk. ~30ms/layer at 1024¬≤. `src/vulca/layers/layered_generate.py:117-124`
- **Cache hit re-validates every time** ‚Äî `ValidationReport` is recomputed even on cache hit. Cache the report alongside the PNG or in a sidecar JSON. `src/vulca/layers/layered_generate.py:146`

### Test coverage gaps
- No e2e test exercises `get_keying_strategy("module.path:callable")` tier-2 path. Add one fixture so the escape hatch can't rot.

### Polish
- `src/vulca/layers/validate.py:73` ‚Äî `int(np.argmax(rows[::-1]))` is subtly confusing; rewrite as `np.where(rows)[0][[0, -1]]`
- `src/vulca/layers/layered_prompt.py:44` ‚Äî f-string with no interpolation
- `src/vulca/layers/layered_generate.py` module docstring claims "Decoupled from the pipeline" but signature couples to the provider's `generate(prompt=, raw_prompt=)` contract ‚Äî narrow type to a Protocol or update docstring
