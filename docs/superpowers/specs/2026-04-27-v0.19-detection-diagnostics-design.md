# v0.19 Spec — Detection diagnostics: silent-drop disclosure + near-miss reporting

**Status**: implementation in flight (codex dispatch 2026-04-27)
**Slug**: `v0.19-detection-diagnostics`
**Authors**: dogfood feedback from IMG_6847 carousel attempt 2026-04-27 (wildflower N=0 + red_car silent drop)
**Cross-ref**: harness plan `~/.claude/plans/showcase-ipad-fancy-clarke.md`; showcase `docs/visual-specs/2026-04-27-ipad-cartoon-roadside/`

## Why

Two real-image dogfood failures surfaced in 30 minutes of carousel work that point at the SAME design pathology: **the orchestrator silently drops entities and reports misleading reasons**.

| Symptom | True mechanism | Misleading report user gets today |
|---|---|---|
| `red_car` missed despite chain reporting `dino:1` | Person-loop rank-pool exhausted: 1 detection, 2 entities, rank=1 ≥ len(yolo_persons)=1 | `reason: "no_detection_after_chain"` + per-entity `attempts: dino:1` (chain-level summary, identical for the entity that DID get assigned the bbox) |
| `wildflower_clusters` N=0 | DINO post-process filters by threshold=0.18 BEFORE we see results; below-threshold matches lost at library boundary | `reason: "dino_not_matched"` (binary fail; cannot tell user "5 candidates at conf 0.10-0.15, lower threshold to capture") |

Both fail in a way the user **cannot diagnose from the report alone**. The user has to read the source to find out why. That's the design gap v0.19 closes.

## Scope (NON-NEGOTIABLE: surgical only)

**In**:
1. Person path missed-record (`scripts/claude_orchestrated_pipeline.py:1086-1098`) — distinguish `chain_returned_zero` from `rank_exceeded_chain_pool`; surface the rank vs pool count.
2. Object path missed-record (`scripts/claude_orchestrated_pipeline.py:1318-1332`) — distinguish `dino_not_matched` (true zero) from `dino_below_threshold` (had candidates) from `dropped_by_within_label_nms` (multiple candidates, NMS kept top).
3. `detect_all_bboxes` (`scripts/claude_orchestrated_pipeline.py:560-624`) — capture below-threshold candidates by lowering the actual DINO call to `min(effective_threshold, NEAR_MISS_FLOOR=0.05)` and partitioning post-DINO results into `assigned` / `near_miss` / `nms_drops`.
4. Tests + changelog + version bump 0.18.0 → 0.19.0.

**Out**:
- Per-entity threshold actually being used during DINO call (`entity.threshold` field is currently ignored — the orchestrator just takes `min(...)`). Identified as the **4th hidden bug** during this analysis. Backlog ticket for v0.20.
- Tiled / upscaled detection paths' near-miss capture. Wrapped to return empty diagnostics for v0.19; full coverage is v0.20+.
- Cross-label dedupe analysis. Within-label NMS is captured; cross-label overlap (e.g. red_car vs yellow_truck physically overlap in image) is NOT v0.19 work.
- Any MCP tool API change. Detection report is a dict embedded in `layers_split` response; new keys are additive.

## Decision points

### D1 — Should `detect_all_bboxes` return shape change?

**Yes — accept the breaking change**.

| Option | Verdict | Reason |
|---|---|---|
| Add `return_diagnostics: bool = False` flag, keep default return shape | ❌ rejected | Two return shapes is API smell; flagged callers vs default callers diverge |
| Wrap with `detect_all_bboxes_v2` returning rich dict | ❌ rejected | Doubles function count; old callers don't benefit |
| **Change return type to dict with 3 keys; update all callers** | ✅ adopted | Single source of truth; callers grep'd in advance: only 2 (`detect_bbox` wrapper + orchestrator); both updated in same commit |

This makes v0.19 a **breaking change** for any external caller of `detect_all_bboxes`. v0.19.0 (semver minor bump justified since detect_all_bboxes is internal API but exposed via `scripts/claude_orchestrated_pipeline.py` — not part of the public src/vulca surface).

### D2 — Where to capture near-miss candidates?

`detect_all_bboxes` only. Tiled/upscaled wrappers return empty `near_miss` / `nms_drops`. Documented limitation.

### D3 — How many candidates to surface per entity?

Cap at **5 per per_entity record** for `near_miss_candidates` and `nms_drop_candidates`, sorted by confidence descending. Capping prevents log bloat when DINO returns 20+ low-conf matches; 5 is enough for the user to spot a pattern (consistent low conf around 0.10 = lower threshold; conf range 0.05-0.30 = pick a few specific ones via `bbox_hint_pct`).

### D4 — Naming convention for the new fields

| Field | When present | Type | Reason |
|---|---|---|---|
| `near_miss_candidates` | object-loop only, when `dino_near_miss[label]` non-empty | `list[{bbox, conf, phrase}]` | Mirrors existing `attempts` shape |
| `nms_drop_candidates` | object-loop only, when within-label NMS dropped non-winners | `list[{bbox, conf, phrase}]` | Same shape |
| `entities_in_chain_pool` | person-loop only, when rank_exceeded | `int` | Numeric for easy filter |
| `this_entity_rank` | person-loop only, when rank_exceeded | `int` | Numeric for easy filter |
| `reason` | always (existing field) | `str` | Refined enum: `chain_returned_zero` / `rank_exceeded_chain_pool` / `dino_not_matched` / `dino_below_threshold` / `dropped_by_within_label_nms` / existing legacy values |

`to_dict()` filters None / empty so absent diagnostics don't pollute records.

### D5 — Multi_instance + near-miss interaction

`multi_instance_no_detection` quality flag fires on `dino_not_matched` (true zero) AND `dino_below_threshold` (because the user requested per-instance and got zero usable). Does NOT fire on `dropped_by_within_label_nms` (NMS keeping ≥1 candidate per multi means at least one passed). Test coverage required.

## Acceptance criteria

1. **red_car case reproduces with new diagnostic**: a synthetic plan with 2 person-class entities + a mock chain returning 1 detection produces a per_entity record for the second entity with `reason: rank_exceeded_chain_pool`, `entities_in_chain_pool: 1`, `this_entity_rank: 1`. New test `test_person_rank_exceeded_chain_pool`.
2. **Wildflower case reproduces with new diagnostic**: a synthetic plan with 1 object entity + a mock `detect_all_bboxes` returning `{"assigned": {}, "near_miss": {label: [(bbox, 0.10, "phrase")]}, "nms_drops": {}}` produces a per_entity record with `reason: dino_below_threshold`, `near_miss_candidates: [{bbox: [...], conf: 0.10, phrase: "..."}]`. New test `test_object_dino_below_threshold`.
3. **Backward compat**: existing tests `tests/vulca/scripts/test_claude_orchestrated_pipeline.py` and `tests/test_layers_v2_split.py` must not red. Pre-existing torch/cv2 failures from memory ignored.
4. **Changelog entry** describing the dogfood event + 2 fixes + the deferred 4th bug (per-entity threshold ignored).
5. **Version**: `pyproject.toml` and `src/vulca/_version.py` (if present) bumped to 0.19.0.

## Risks

- **R1 (low/medium)**: lowering DINO threshold to 0.05 increases label-match loop work proportionally to candidate count. Worst case ~10× more candidates to phrase-match. Mitigated by: phrase-matching is O(N×L) where L=label count, both small constants.
- **R2 (low/high)**: `detect_bbox` wrapper consumers downstream might use return shape. Codex is instructed to grep all callers and update. Verification step lists call sites in final report.
- **R3 (low/low)**: tiled/upscaled wrappers don't get diagnostics in v0.19. Documented as TODO; user just sees `dino_not_matched` for those paths (current behavior preserved).
