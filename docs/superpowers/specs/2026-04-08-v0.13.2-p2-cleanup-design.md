## Design decisions (resolved during brainstorm)

| Decision | Resolution | Rationale |
|---|---|---|
| Task 17 retry budget | **Implement in-process retry budget=2** (Q2: A) | Fulfills plan promise; transient provider errors self-heal; `attempts` telemetry stops lying |
| Retry scope | Only `generation_failed`, never `validation_failed` | Validation is deterministic — retry is pure cost |
| Retry backoff | **Full jitter**: `random.uniform(0, 0.5 * 2**attempt)` (Q5: b) | Shortest code, AWS-recommended, best concurrent-retry waveform; monkeypatchable in tests |
| `_provider_supports_native` replacement | **Capability frozenset** `ClassVar[frozenset[str]]` (Q3: C) | Room for future capabilities (`streaming`, `batch`) without Protocol churn |
| Cache performance | **Sidecar `<hash>.report.json`** + re-read disk bytes for cache write (Q4: A) | PNG stays directly openable for debug; 30ms/layer @ 1024² reclaimed; one-line fix for double-encode |
| `LayerFailure.reason` type | **`Literal["generation_failed", "validation_failed"]`** | Zero runtime cost, typechecker-covered |
| `layer_extras` key collision | **Whitelist spread** — only allow known keys, raise on unknown | No schema change needed |
| `layered_cache.py` temp-write-and-rename | **All cache writes** go through `tempfile + os.replace` | Uniform, no flag plumbing, MCP-safe by default |

## Work breakdown — 5 groups, TDD per item

### Group 1 — Correctness hardening

1. **`_apply_alpha` silent resize → assert** (`src/vulca/layers/layered_generate.py:83-88`)
   - Replace bilinear-fixup with `assert mask.shape[:2] == img.shape[:2], f"..."`
   - Test: feeding mismatched shapes raises `AssertionError`
2. **`layered_cache.py` temp-write-and-rename** (`src/vulca/layers/layered_cache.py:44-51`)
   - All `.png` writes go `tempfile.NamedTemporaryFile(dir=cache_dir, delete=False)` → `os.replace`
   - Test: simulate partial write (write bytes + kill) leaves cache unchanged
3. **Tier-2 keying loader error wrapping** (`src/vulca/layers/keying/__init__.py:41-47`)
   - `ModuleNotFoundError` / `AttributeError` → `ValueError(f"Unknown keying strategy {spec!r}: {orig}")`
   - Test: typo like `"vulca.foo:bar"` raises `ValueError` with helpful message

### Group 2 — Retry & telemetry *(design-level codex review before merge)*

4. **In-process retry budget in `generate_one_layer`** (`src/vulca/layers/layered_generate.py:138,151,156`)
   - Budget = 2 retries (3 total attempts)
   - Retry only on `generation_failed` (provider raised); validation failures fall through immediately
   - Backoff: `await asyncio.sleep(random.uniform(0, 0.5 * (2 ** attempt)))` before retry
   - `attempts` field reports real count
   - Tests:
     - Transient `generation_failed` → succeeds on attempt 2, `attempts=2`
     - Persistent `generation_failed` → fails after 3 attempts, `attempts=3`
     - `validation_failed` → 1 attempt, no retry, `attempts=1`
     - Backoff sleep happens between attempts (monkeypatch `random.uniform` + `asyncio.sleep`)
5. **Plan doc sync** — update `docs/superpowers/plans/2026-04-08-layers-redesign.md` Task 17 to reflect reality ("retry budget=2 in-process, plus external `layers retry` CLI for operator-driven full regen")
6. **`LayerFailure.reason` → `Literal`** (`src/vulca/layers/layered_generate.py:197-207`)
   - `from typing import Literal`; type narrows across `LayerFailure`, CLI string match, MCP pass-through
   - No runtime code change expected; typechecker validation only

### Group 3 — Provider capability surface

7. **Add `capabilities: ClassVar[frozenset[str]]`** to provider base/Protocol
8. **Mock provider** declares `frozenset({"raw_rgba"})`; other providers get `frozenset()` default
9. **`_provider_supports_native`** (`src/vulca/pipeline/nodes/layer_generate.py:83-86`) replaced with `"raw_rgba" in getattr(provider, "capabilities", frozenset())`
   - Test: custom mock class `mock_v2` with `capabilities = frozenset({"raw_rgba"})` routes through native path; legacy string match would have missed it

### Group 4 — Cache performance

10. **Sidecar `<hash>.report.json`** — write `ValidationReport.model_dump_json()` next to PNG on cache store
11. **Cache hit reads sidecar** — `layered_generate` cache-hit path loads `ValidationReport` from JSON, skips `validate_layer` call
    - Missing sidecar on a legacy cache entry → fall back to re-validate + write sidecar (auto-migration)
    - Tests: cache hit with sidecar skips validation (assert `validate_layer` not called); cache hit without sidecar re-validates and writes one
12. **Cache write reuses disk bytes** (`src/vulca/layers/layered_generate.py:117-124`)
    - After writing PNG to output dir, re-read file bytes for cache write instead of PIL→bytes re-encode
    - Test: assert PIL `.save()` called exactly once per layer (not twice)

### Group 5 — Manifest & retry polish

13. **`layer_extras` whitelist spread** (`src/vulca/layers/manifest.py:51-53`)
    - Define `_ALLOWED_EXTRAS: frozenset[str] = frozenset({...})` with current known keys
    - Unknown key in `layer_extras` → `ValueError`
    - Test: unknown key raises; known keys merge correctly
14. **Retry preserves non-validation warnings** (`src/vulca/layers/retry.py:210-222`)
    - Read `warnings` from prior manifest, merge into rebuilt manifest (e.g. tradition-layerability discouraged warnings)
    - Test: manifest with warning → retry → rebuilt manifest still has warning
15. **Polish nits** — bundle remaining ~3 minor items from the P2 backlog (renaming, typos, docstring gaps). Decided inline during implementation: trivial-and-worth-it → do; bikeshed → drop.

## Risks

- **Group 2 retry** has real behavioral change (provider cost 3× worst case). Mitigated by: retry only on `generation_failed`, `validation_failed` never retries, design-level codex review gate before merge.
- **Group 4 sidecar auto-migration** — legacy cache entries without sidecar fall back to re-validate. First run post-upgrade is no faster; subsequent runs are. Acceptable.
- **Group 5 whitelist** is a breaking change if any caller passes unknown `layer_extras` keys. Mitigated by: grep first to confirm no current caller does, add all known keys to whitelist, raise with a clear error listing allowed keys.
