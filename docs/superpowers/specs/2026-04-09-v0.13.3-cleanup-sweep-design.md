# v0.13.3 Cleanup Sweep — Design

**Date:** 2026-04-09
**Status:** Approved, ready for implementation plan
**Scope:** v0.13.3 candidate backlog items 1, 2, 3, 5. Items 4 (live-API tests), 6 (mypy gate), 7 (cv2 baseline) deferred to separate sessions — each is its own infrastructure decision.

## Background

v0.13.2 shipped 2026-04-08 (tag `v0.13.2`, commit `1c5d632`). Ship retro identified a cleanup backlog captured in memory file `project_v0_13_3_candidates.md`. v0.13.3 is a focused cleanup sweep in the `layers/` + provider-capability area. Items 4/6/7 (infrastructure decisions) were deliberately excluded to keep scope tight.

## Goals

1. Eliminate the remaining `provider == "mock"` literals at 4 pipeline sites where future `mock_v2` or user-registered mock variants would misroute.
2. Reconcile the `ImageProvider.capabilities` Protocol attribute with its defensive `getattr` fallback — pick a single contract model.
3. Reduce `generate_one_layer` branch count by extracting the two load-bearing subsections (retry loop, sidecar report resolution) into independently testable helpers.
4. Close the sidecar round-trip coverage gap left by v0.13.2 T9.

## Non-Goals

- No new feature work.
- No changes to provider registration, cache file format, or validation semantics.
- Backlog item 4 (live-API tests), 6 (mypy/pyright gate), 7 (cv2 baseline failures) are **out of scope**. Each needs its own brainstorm.
- No rewrite of `generate_one_layer` into the full 3-way split from the backlog — that would require a state dataclass and is more scope than this sweep earns.

## Design

### 1. New module: `src/vulca/providers/capabilities.py`

A coarse name-keyed capability table, distinct from the existing `ImageProvider.capabilities` Protocol attribute.

**Separation of concerns:**
- `ImageProvider.capabilities` (Protocol attr) → image-runtime details (`raw_rgba`, future `streaming`, `batch`, `reference_image`). Scoped to ImageProvider instances.
- `capabilities.py` (new) → coarse "what kinds of work does this provider name do at all" — answers questions asked by pipeline nodes that only hold a string provider name and may not have a typed provider object at all (e.g. `plan_layers.py` uses litellm directly).

**Sketch:**

```python
"""Coarse provider-kind capability lookup (by provider name).

Distinct from ImageProvider.capabilities — that Protocol attribute is
scoped to image-runtime details (raw_rgba, streaming, ...). This module
answers the coarser question "what kinds of work does a provider named X
do at all", needed by pipeline nodes that only hold a string name.
"""
from __future__ import annotations

VLM_SCORING  = "vlm_scoring"
LLM_TEXT     = "llm_text"
COST_TRACKED = "cost_tracked"

_DEFAULT_CAPABILITIES = frozenset({VLM_SCORING, LLM_TEXT, COST_TRACKED})

_PROVIDER_CAPABILITIES: dict[str, frozenset[str]] = {
    "mock": frozenset(),
    # Future explicitly-limited providers declare reduced caps here.
    # e.g. "sd_local": frozenset() — image-only, free, no VLM/LLM.
}

def provider_capabilities(name: str | None) -> frozenset[str]:
    """Return coarse capabilities for a provider name.

    Unknown/unregistered names return the FULL capability set — preserves
    pre-v0.13.3 behavior where every non-mock provider went through real
    code paths and was cost-charged via _COST_PER_IMAGE.get(name, 0.067).

    None/empty → frozenset() (same as mock — no real work).
    """
    if not name:
        return frozenset()
    return _PROVIDER_CAPABILITIES.get(name, _DEFAULT_CAPABILITIES)
```

**Vocabulary rationale:** three narrow capabilities (option A from the brainstorm) match three actual questions at the 4 sites. No inference, no coupling between "has any real capability" and "costs money" — that coupling would break the moment we add a free real provider (local Stable Diffusion, etc.).

**Unknown-name behavior (CORRECTED from initial draft):** returns the **full** capability set. This preserves the existing implicit contract where `provider != "mock"` meant "real provider — charge it, call real VLM, call real LLM." Only `"mock"` (and future explicitly-limited providers listed in the table) opt out. The table enumerates restrictions, not grants. Rationale: the original "empty = conservative" default would have silently downgraded unknown providers (including `nb2`, `replicate`, any user-registered name) to mock paths — a behavior-breaking regression.

**`src/vulca/pipeline/nodes/generate.py:72`** — **unchanged behavior**, add explanatory comment:

```python
async def run(self, ctx: NodeContext) -> dict[str, Any]:
    provider = ctx.provider or "mock"

    # Intentional literal: "mock" here means "no registered image provider
    # AND no injected instance" — the short-circuit to canned mock output.
    # Registered mock_v2 / user-defined providers are routed through the
    # ImageProviderRegistry via _provider_generate and never hit this branch,
    # so a capability lookup here would be functionally redundant.
    if provider == "mock" and ctx.image_provider is None:
        return self._mock_generate(ctx)

    return await self._provider_generate(ctx, provider)
```

Rationale: this site is already structurally correct. A registered `mock_v2` lands in `_provider_generate` (registry path); the literal only fires when the user explicitly names `"mock"` and provides no instance override. Adding a capability lookup would be ceremony without a behavior change.

### 4. `generate_one_layer` split (backlog item 3, option b)

`src/vulca/layers/layered_generate.py:139`

Two private helpers extracted. Main function drops from ~137 lines to ~80.

**`_call_provider_with_retry(provider, prompt, layer_name) -> tuple[bytes, int]`**

- Wraps current lines 189–213 (retry loop).
- Signature: provider, prompt, layer name (for logging); returns `(rgb_bytes, attempts)` on success.
- On exhausted budget: raises the last captured exception. Caller (`generate_one_layer`) catches it and returns `LayerOutcome(ok=False, ..., attempts=...)`. Raising was chosen over a sentinel tuple because it keeps the happy path single-valued.
- **Preserves exception taxonomy exactly:** `AssertionError`, `TypeError`, `asyncio.CancelledError` propagate without being caught (programmer bugs / control flow, not transient failures). This invariant is load-bearing — v0.13.2 G2 review nit — and must not drift in the refactor.
- Retry budget constant `_RETRY_BUDGET` stays module-level.
- Backoff: `random.uniform(0, 0.5 * (2 ** attempt))` preserved.

Indirect benefit: the retry loop becomes unit-testable against controlled fake exceptions without needing the full layer pipeline. This partially addresses backlog item 4 (no live-API retry coverage) at the unit level — does not replace a future live-API integration test, but raises confidence that the retry state machine itself is correct.

**`_obtain_validation_report(cache_hit, cache, cache_key, out_path, position, coverage, cache_put_ok) -> ValidationReport`**

- Wraps current lines 235–264 (three-state sidecar logic).
- Three branches preserved:
  1. **Cache hit with valid sidecar** → reuse persisted report via `_report_from_dict`.
  2. **Cache hit, sidecar missing or schema-stale** → re-validate from disk, write fresh sidecar (legacy cache migration path, debug log for observability).
  3. **Fresh generation** → validate from disk, write sidecar.
- **Orphan sidecar guard** (v0.13.2 G4 invariant) must stay intact: `cache.put_report` is called only when `cache_put_ok` is True. Violation = stale sidecar pointing at missing PNG body.
- Best-effort `cache.put_report` exception handling preserved (broad `except Exception`, warning log).

**Main function control flow after split:**

1. Build prompt + cache key (unchanged, lines 154–171).
2. Cache body lookup (unchanged, lines 173–183).
3. If no cache hit: call `_call_provider_with_retry`, handle exception → `LayerOutcome(ok=False)`, else key + save + `cache.put` (best-effort).
4. Call `_obtain_validation_report`.
5. Assemble `LayerOutcome` (unchanged, lines 266–275).

Reads top-to-bottom with no nested three-state logic inline.

### 5. Sidecar round-trip test (backlog item 5)

**Location:** `tests/vulca/layers/test_layer_cache_sidecar.py` (existing file if present; otherwise new). Implementation step must read the existing test layout before deciding.

**Primary test** — asserts full field coverage survives JSON round-trip:

```python
def test_validation_report_round_trip_preserves_all_fields():
    original = ValidationReport(
        ok=False,
        warnings=[
            ValidationWarning(kind="coverage", message="low",
                              detail={"actual": 0.12}),
            ValidationWarning(kind="position", message="off", detail=None),
        ],
        coverage_actual=0.12,
        position_iou=0.45,
    )
    as_dict = _report_to_dict(original)
    restored = _report_from_dict(json.loads(json.dumps(as_dict)))

    assert restored.ok == original.ok
    assert restored.coverage_actual == original.coverage_actual
    assert restored.position_iou == original.position_iou
    assert len(restored.warnings) == len(original.warnings)
    for r, o in zip(restored.warnings, original.warnings):
        assert isinstance(r, ValidationWarning)
        assert r.kind == o.kind
        assert r.message == o.message
        assert r.detail == o.detail
```

Locks the contract v0.13.2 T9 established: every `ValidationReport` field, including nested `ValidationWarning` instance type, must survive `_report_to_dict` → JSON → `_report_from_dict`.

**Secondary test** — documents `_report_from_dict` behavior on unknown fields. **Decision deferred to implementation:** read the current `_report_from_dict` body first. If it silently drops unknown fields, add a test asserting that (documents it as intentional — forward compat with future schema additions). If it raises, don't add the test; the strictness is already load-bearing.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| `_COST_PER_IMAGE` table and `COST_TRACKED` capability set drift — provider in cost table but explicitly declared without `COST_TRACKED` → cost gate skips charging | Invariant test: any provider explicitly listed in `_PROVIDER_CAPABILITIES` without `COST_TRACKED` must NOT appear in `_COST_PER_IMAGE`. Unknown names are safe (default includes `COST_TRACKED`). |
| Retry loop extraction accidentally catches `AssertionError`/`TypeError`/`CancelledError` | Explicit test cases for each; review checkpoint called out in design §4 |
| `_obtain_validation_report` refactor breaks the orphan-sidecar guard (v0.13.2 G4) | Dedicated branch-coverage test with `cache_put_ok=False` |
| Unknown provider name gets full capabilities (inverted default) — typo in provider name silently routes to real paths + charges money | Acceptable: matches pre-v0.13.3 behavior exactly. Stricter handling (raise on unknown) is its own discussion — not a regression. |
| Capability vocabulary grows ad-hoc | Three constants start as module-level symbols; future caps added with an explicit comment explaining the axis they represent. |
