# Source Summaries

## GEO / Figma Slides Case Study

URL: https://geo-nyc.com/projects/figma-slides/

GEO describes the Figma Slides launch work as a product-facing creative system rather than a single campaign image. The useful lesson for Run 1 is that a presentation product needs a confident launch surface, a range of reusable starting points, and enough aesthetic variety to help new users understand the tool's expressive range. The Run 1 deck should translate that idea into original, editable slide patterns instead of copying any template, screenshot, or product-specific look.

## Config 2024 in Review

URL: https://www.figma.com/blog/config-2024-recap/

The Config 2024 launch context connects AI, redesigned product surfaces, developer workflow, and Figma Slides into one event story about how product teams work. The relevant lesson is sequencing: the deck should frame a broad shift in work, then introduce the presentation product as a practical answer for collaboration, design control, and communication. Run 1 should not mimic Figma's event visuals or language; it should use the launch-event structure as a reference for original narrative planning.

## Config 2025 Visual Identity

URL: https://www.figma.com/blog/how-we-shaped-the-visual-identity-for-config-2025/

Figma's Config 2025 identity article is useful because it explains a system that stretches across surfaces while staying grounded in a flexible visual concept. The Run 1 lesson is to build a small design vocabulary that can vary by slide role: primitives, motion cues translated into static rhythm, sparse celebratory marks, and utility-first hierarchy for dense information. The deck should use its own simple system tokens and native PowerPoint shapes, not Figma glyphs, event assets, or brand marks.

## MUSE Creatives / Supervity AI Keynote Presentation Case

URL: https://musecreatives.org/case-studies/visual-presentation-for-ai-thought-leadership/

The Supervity AI case frames a common commercial problem: a technical AI story needs to become a clear keynote narrative with stronger hierarchy, clearer visual flow, and proof-oriented diagrams. The useful lesson for Run 1 is to make AI capability legible quickly by turning dense process claims into steps, proof objects, and labeled visual moments. Run 1 should keep that communication strategy while avoiding the case study's visuals, palette, layout, and generated assets.

## Use Boundary

These references are allowed for reference analysis only. Use them to write original design notes, narrative rules, style tokens, and code-generation constraints. Do not copy proprietary visuals, screenshots, layouts, brand marks, template files, glyphs, full prose, long quotes, or any source asset into the generated deck or committed pack.
