# Design Notes

## Launch Identity

The launch identity should feel like a disciplined product system, not a decorative event poster. Use a compact vocabulary of primitives, panels, proof markers, and directional connectors so every slide feels connected while still changing shape by narrative role. The reference lesson is system stretch: a few reusable marks can support cover impact, workflow explanation, comparison, and closing without copying any outside identity.

## AI Explainability

AI claims should be shown as workflow control rather than abstract intelligence. Use visible inputs, transformations, QA gates, and output artifacts. The audience should see what the model or agent is allowed to do, what stays deterministic, and where human review enters the loop. Avoid vague magic language and use diagrams with concrete labels.

## Modular Slide Systems

Each slide pattern should have a clear job: establish the promise, name the problem, show the reference shift, explain the workflow, expose design memory, prove code generation, show the review loop, compare outcomes, make a product decision, and close with a next action. These patterns should share spacing, typography, and signal colors so a code generator can produce consistency without requiring manual art direction.

## Editability

Editability is a product feature, not an export detail. Titles, claims, labels, matrices, diagrams, connectors, and status badges must stay editable in PowerPoint. Bitmap images, if any, should support atmosphere only and must never contain essential text. Layout JSON should provide a review surface for clipping, overlap, and slide-to-slide consistency.
