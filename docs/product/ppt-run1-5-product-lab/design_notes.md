# Design Notes

## Brief Cockpit

The Brief Cockpit is the first product surface. It exposes audience, business goal, must-win decision, constraints, desired tone, output promise, and blocked publish status in one slide-level control panel.

## Learning Map

The Learning Map turns tutorial notes and source observations into an editable path: evidence enters, rules are selected, design memory fields are compiled, primitives are chosen, and QA signals are attached.

## Experiment Lab

The Experiment Lab compares prompt-only, full Vulca, and bad-data arms as blocking runs. It should feel like a controlled product bench with arm status, input boundaries, expected failure modes, and a visible public release gate.

## Layout Controls

Use one dominant proof object per slide, no more than three secondary panels, readable main claims from a contact sheet, native editable structures, and clear not-run status wherever evidence has not been generated.
