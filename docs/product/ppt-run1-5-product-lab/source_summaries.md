# Source Summaries

Reference-analysis-only boundary: these notes translate public reference observations into original product-lab constraints. Do not copy reference visuals, screenshots, layouts, logos, article text, template designs, glyph systems, or proprietary creative assets.

## geo_figma_slides

The Figma Slides project shows presentation design as an onboarding surface, not just a finished artifact. Its useful lesson for this pack is that a deck system can help new users understand a tool by giving them ready-to-edit structures across multiple use cases, so the Vulca deck should demonstrate editable proof objects rather than static polish.

## figma_config_2024

The Config 2024 recap frames Figma Slides, AI features, UI changes, Dev Mode, and quality-of-life updates as connected product surfaces. For this pack, the reference supports a progression where a launch deck moves from workflow pain to product control, then to proof that design and code can stay close through status, component, and QA surfaces.

## figma_config_2025_identity

The Config 2025 identity story emphasizes a flexible visual concept that can stretch across many surfaces while retaining a coherent system. The Run 1.5 lesson is to encode adaptable primitives and layout controls in design memory, so the generator has a reusable language instead of a one-off decorative style.

## supervity_ai_keynote

The Supervity presentation case study is useful because it describes a commercial AI deck moving from dense technical material into a clearer narrative, stronger hierarchy, and digestible visual steps. This pack uses that observation as a boundary for commercial clarity: every generated slide should expose one decision or proof object quickly enough for a live business audience.
