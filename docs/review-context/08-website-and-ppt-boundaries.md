# Website And PPT Boundaries

Vault status: product boundary memory.

## Website Role

The website is the public trust and acquisition surface. It should answer:

- What is VULCA?
- Why does AI-native visual work need review, evidence, and release control?
- What is a Creative Repo?
- Why is VULCA credible?
- How does a visitor request a pilot, partnership, investor brief, or research
  path?

The website should not expose internal run IDs or raw proof logs as the first
story.

Current website direction:

- commercial homepage
- film-led hero
- customer rail: Brand, Museum, Agency, Tender, Education, Research
- proof and release-gate language
- bilingual content

## PPT Proof Lab Role

The PPT branch is an internal proof lab for usecase-to-deck workflows.

It contains valuable:

- source boundary rules
- deck briefs
- claim spines
- visual briefs
- generation prompts
- QA prompts
- visual quality evaluations
- public-blocked gates

It is not yet a public product claim that VULCA produces polished decks.

## Current PPT Boundary

- public release remains blocked

Therefore the PPT work can be used as:

## Integration Rule

The website may reference the PPT proof pipeline only as internal proof or a
guarded evidence example.

Allowed:

> Internal proof pipeline: critique -> motif contract -> visual primitive ->
> comparison -> review gate.

Required boundary:

> Public release is blocked until visual quality evaluation passes.
