\section{团队与顾问}
\label{sec:team}

\subsection{团队画像（当前）}

Vulca 当前以\keyword{solo-maintainer}模式高速推进，
从 2026-04-09（v0.14.0）至 2026-04-23（v0.17.5）共发布\keyword{11 个正式版本}（PyPI 可核验），
平均每 1.3 天一次 ship，累计 21 个 MCP 工具、3 个端到端 shipped skill、高覆盖率测试套件。
这种迭代密度在天使阶段的\keyword{一人项目}里是\keyword{极少见}的，直接反映了创始人的\keyword{工程密度与产品判断力}。

\subsection{核心成员}

\subsubsection*{创始人 / 技术负责人：Yu Haorui}

\keyword{博士研究方向与产品底盘同源}\quad
现为英国 \keyword{University of Dundee}（邓迪大学）\keyword{Duncan of Jordanstone College of Art \& Design}（DJCAD）在读博士研究生，
师从 \keyword{Prof.~Natasha Lushetich}（DJCAD 跨学科研究方向），
与 Computing 系 \keyword{Dr.~Ramon Ruiz-Dolz}（计算论证 / NLP 方向，ELLIS Society 成员）密切合作。
博士课题 \textit{"Evaluating Vision-Language Models for Cultural Understanding in Heritage and Arts Contexts"}
聚焦\keyword{视觉-语言模型在艺术与非遗语境中的文化偏见评估与修正}——正是 Vulca 产品的学术底盘。

\keyword{已发表 / 在投的学术成果}：
\begin{itemize}
  \item \keyword{EMNLP 2025 Findings（一作）}：\textit{A Structured Framework for Evaluating and Enhancing Interpretive Capabilities of Multimodal LLMs in Culturally Situated Tasks}。
  提出 L1–L5 分层评估框架（VULCA），已被纳入 Vulca SDK 的\keyword{文化评估核心模块}。
  与 Ramon Ruiz-Dolz、Qiufeng Yi 合作，论文 pp.~1945–1971。
  \item \keyword{VULCA-Bench（一作，arXiv \href{https://arxiv.org/abs/2601.07986}{2601.07986}）}：多模态文化评估基准集，\keyword{目前 ACL 2026 Dataset Track 审稿中}。arXiv 公开版本记录的是 8 种艺术传统、7410 样本的\keyword{阶段性快照}；Vulca SDK 当前版本（v0.17.5）已扩展至 \keyword{13 种 YAML 定义}（含通用基线），论文下一版将同步更新——这一差异是"论文记录一个时间点 vs 产品持续迭代"的正常现象。
  \item \keyword{Art Critique（arXiv \href{https://arxiv.org/abs/2601.07984}{2601.07984}）}：\textit{Cross-Cultural Expert-Level Art Critique Evaluation with Vision-Language Models}，与 Xuehang Wen、Fengrui Zhang、Qiufeng Yi 合作。
\end{itemize}

\keyword{工程能力证据}\quad 在 14 天内独立完成 Vulca 从 v0.14 到 v0.17.5 共 11 个正式版本的迭代，
平均每 1.3 天一次 ship；累计 21 个 MCP 工具、3 个端到端 skill、高覆盖率测试套件。
工程流程（brainstorm $\to$ spec $\to$ plan $\to$ ship-gate）本身开源在仓库的 \code{docs/superpowers/} 目录，
保证团队扩张时的\keyword{上下文可继承性}——这是从学术能力到产品执行力的关键桥梁。

\keyword{为何做 Vulca}\quad 博士课题揭示了当前多模态 LLM 在文化语境下的系统性偏差（特别是中国艺术传统），
但学术论文只能\keyword{提出评估框架}，无法\keyword{部署到真实创作工作流}。
Vulca 的使命是把 L1–L5 文化评估 + agent-native 执行能力\keyword{从论文里搬进真实的博物馆、出版社、设计工作室}，
让"AI 理解中国艺术"从 benchmark 变成可交付产品。

\subsection{文化与协作原则（自证合规性）}

\begin{itemize}
  \item \keyword{开源优先}\quad 核心 SDK 与 MCP 工具集保持 Apache 2.0，商业化组件与服务层单独；
  \item \keyword{ship-first}\quad 每个新特性都走"brainstorm $\to$ spec $\to$ plan $\to$ ship gate"的标准流程，所有 gate 通过才发布（流程本身开源于仓库的\code{docs/superpowers/}目录）；
  \item \keyword{learning-centric}\quad 所有关键决策记录在可追溯的\code{memory}与\code{specs}中，保证团队扩张时的\keyword{上下文可继承性}；
  \item \keyword{跨文化意识}\quad 13 种艺术传统的 YAML 评估体系本身就是文化纵深积累的一部分，团队鼓励补充新传统、邀请社区贡献。
\end{itemize}
